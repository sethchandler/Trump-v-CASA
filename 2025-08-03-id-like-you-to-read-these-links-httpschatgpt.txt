  ## How the agent will read from GitHub

  * **Plain-text & small binary files (≈ 10 MB or less):**
    The agent can `GET` any raw URL, e.g.
    `https://raw.githubusercontent.com/your-user/your-repo/main/opinions/24a884.pdf`.
    That covers PDFs, HTML, Markdown, JSON, images, even short audio clips.

  * **Large binaries (big MP3s, > 10 MB):**

    1. Commit them with **Git LFS** so GitHub serves them via the LFS CDN.
    2. Or skip committing and just store the link (the SCOTUS MP3 is streamable in place).
       Either way the website can embed the remote URL; the agent only needs the metadata.

  * **Rate limits:** the web-reading tool pulls over HTTPS like any browser, so public repos are fine. Private repos won’t work unless you add a public
   raw link (e.g., GitHub Pages or an S3 presigned URL).

  * **Suggested repo layout**

  ```
  trump-v-casa-site/
  │

  │   └─ 24a884.pdf
  ├─ briefs/
  ├─ lower-courts/

  ├─ statutes/
  ├─ scholarship/
  └─ assets/
  ```
